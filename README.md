# 🚜 Бухоро Агрокластер — Транспорт ҳисоботи v2

Кунлик транспорт ишлари ҳисоботини автоматлаштириш тизими.
Кўп фойдаланувчили версия — авторизация, роллар, серверда ишлайди.

---

## Имкониятлар

- **Авторизация**: Логин/парол билан кириш
- **3 та роль**: Админ → Оператор → Наблюдатель
- **Мулти-тенант**: Ҳар бир оператор фақат ўзига тегишли ташкилотларни кўради
- **Кунлик маълумот киритиш**: Техника, иш тури, буюртмачи, миқдор, нарх, тўлов
- **Excel ҳисобот**: 5 та варақли ҳисобот (аслиятга мос)
- **Справочниклар**: Ташкилотлар, техникалар, иш турлари, буюртмачилар
- **Аудит**: Ким, қачон маълумот киритгани сақланади

## Ўрнатиш (Windows Server)

### 1-қадам: Python ўрнатиш
- [python.org](https://python.org) дан Python 3.10+ юклаб ўрнатинг
- Ўрнатишда **"Add to PATH"** белгиланг

### 2-қадам: Папкани кўчириш
`transport-report` папкани серверга кўчиринг, масалан `C:\transport-report\`

### 3-қадам: Биринчи ишга тушириш (тест)
```
cd C:\transport-report
pip install -r requirements.txt
python init_data.py
python run_server.py
```
Браузерда `http://localhost:5050` очинг.
Логин: `admin` / Парол: `admin123`

### 4-қадам: Windows сервис сифатида ўрнатиш (24/7 ишлаши учун)

1. [NSSM](https://nssm.cc/download) юклаб олинг
2. `nssm.exe` ни `C:\transport-report\` папкага қўйинг
3. `install_service.bat` ни **администратор сифатида** ишга туширинг

Сервис автоматик ишга тушади ва сервер қайта юкланганда ҳам ишлайверади.

### 5-қадам: Ташқи доступ (интернет орқали)

Сервер IP-адреси бўйича кириш (масалан `http://10.103.25.200:5050`):
- Windows Firewall да 5050-портни очинг
- Роутерда port forwarding қилинг (агар ташқаридан керак бўлса)

Домен бўйича кириш (масалан `http://transport.agrocluster.uz`):
- DNS да домен → сервер IP га йўналтиринг
- (Опционал) Nginx ёки IIS орқали reverse proxy + SSL

## Бошқариш

### Фойдаланувчи қўшиш:
1. `admin` бўлиб киринг
2. **Админ → Фойдаланувчилар** саҳифасига ўтинг
3. Логин, парол, роль, ташкилотларни танланг

### Сервисни бошқариш (CMD администратор):
```
nssm start TransportReport     — ишга тушириш
nssm stop TransportReport      — тўхтатиш
nssm restart TransportReport   — қайта ишга тушириш
nssm remove TransportReport confirm — ўчириш
```
